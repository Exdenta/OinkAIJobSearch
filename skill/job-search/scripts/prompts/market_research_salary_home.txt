OUTPUT STYLE — caveman english
==============================
For every natural-language field you emit (summaries, "why" fields,
descriptions, focus notes, narrative paragraphs, body paragraphs):
drop articles (a/an/the), filler ("just", "really", "actually",
"simply", "basically", "essentially"), pleasantries (no "of course",
"happy to", "certainly"), and hedging. Fragments OK. Use short
synonyms ("fix" not "implement a solution for"; "big" not
"extensive"; "use" not "leverage"; "show" not "demonstrate").
Technical terms stay exact.

DO NOT apply this style to:
- numbers, code, URLs, JSON keys, enum values, schema field names,
  IDs, hashes
- quoted error messages
- legal / privacy / consent text
- raw extracted titles, company names, location strings, or any
  value parsed downstream by another system
- bullet lists where each item is one short noun phrase (already
  terse — leave alone)

Keep meaning, drop fluff. If the rule conflicts with grammatical
correctness (e.g. titles of works, proper nouns), correctness wins.

DERIVATION NOTES
================
- `currency_local`: ISO 4217 code (EUR, USD, GBP, CAD, AUD, PLN, CHF,
  SEK, NOK, DKK, CZK, HUF, RON, TRY, BRL, MXN, INR, SGD, JPY, KRW, …).
  Pick the one actually paid in `{location}`.
- `bands`: one entry per level. Cover the full set
  {"junior", "mid", "senior", "lead", "staff"} even when demand at an edge
  is thin — if you lack data for a level, still produce the entry with
  your best integer estimates and mention the thinness in `narrative`.
  Each entry:
    `{"level": "<junior|mid|senior|lead|staff>",
      "p25_local": <int>, "p50_local": <int>, "p75_local": <int>,
      "p25_usd":   <int>, "p50_usd":   <int>, "p75_usd":   <int>}`.
  All integers. No decimals. Annualized gross base salary unless the
  region customarily reports packages differently (explain in
  `total_comp_notes` if so).
- `total_comp_notes`: plain-text string. Cover: bonus customs (STI %
  bands), equity/RSU prevalence (none / occasional / common / heavy),
  13th-month pay, pension/employer contributions, signing bonus norms,
  relocation norms, taxation headline (gross vs net clarification).
  Also state the FX rate used and its date.
- `narrative`: 2-4 plain-text sentences. Tell the candidate where in the
  bands a `{years_experience}`-year `{primary_role}` with `{stack_primary}`
  realistically sits, and any split between MNC-pay vs local-startup-pay.
- `confidence`: "high" if ≥3 independent reputable salary sources
  converge within 15% at p50 for the candidate's target level; "medium"
  if wider spread; "low" if you had to extrapolate from adjacent roles
  or adjacent markets.
